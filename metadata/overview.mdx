Metadata can be used to reference stored objects that can build up sections of the pipeline.

```json
"metadata": {
    "project_id": "0273e50f-088d-4d56-b3f3-2e39d4293ff5",
    "device_fingerprint_id": "e79707cd-6b8f-4328-bd35-e3ebdf7a0bea",
    "geo_fingerprint_id": "9fa06120-eb07-4426-913b-1657b28efd45",
    "template_id": "0947c7fb-c13d-4a77-82fc-246cce01307c"
}
```

The following:
- **[Project](/api-reference/endpoint/project)** 
- **[Device Fingerprint](/pipeline/scraping/fingerprint#device-fingerprint)** 
- **[Geo Fingerprint](/pipeline/scraping/fingerprint#geo-fingerprint)** 
- **[Template](/metadata/templates)**

Can be referenced by their assigned ID. The user must be associated with the same organization **except for when using a Geo Fingerprint**.
